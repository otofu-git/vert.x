vert.x

최신 웹과 기업을 위한 매우 쉬운 비동기식 어플리케이션 개발

아직 한국어 번역이 진행중입니다. 덜 된 부분이 많네요. 잠시만 기다려 주세요. ^^

다양한 언어


여러분의 어플리케이션 구성요소들을 JavaScript, CoffeeScript, Ruby, Python, Groovy 혹은 Java로 작성해보세요. 또는 하나의 어플리케이션에서 여러 언어들을 같이 사용할 수도 있습니다.

간단함


지나치게 단순하지는 않습니다. 몇줄만의 코드로 확장성 있는 어플리케이션을 제작할 수 있습니다. 정신없는 xml 설정은 필요없습니다.


확장성


메시지 전달을 통해 확장하고 불변 공유 데이터를 이용해 서버의 코어들을 효율적으로 활용합니다.



동시성


굉장히 간단한 동시성 모델로 기존의 멀티-스레드 프로그래밍의 번거로움으로 부터 벗어날 수 있습니다.



더 알아보세요...

특징


  • 분산 이벤트 버스가 서버사이드와 클라이언트 전반에 걸쳐 있어 매우 쉽게 실시간 웹을 구현할 수 있습니다.

  • 이벤트 버스 모듈에는 웹 서버, persistor, mailer 그리고 work queue 등이 있습니다.

  • TCP/SSL 서버와 클라이언트

  • HTTP/HTTPS 서버와 클라이언트

  • WebSockets 지원

  • SockJS 지원

  • 모듈 시스템

  • 다양한 문서

  • 오픈 소스이며 코드는 ASL 2.0 라이센스를 따릅니다.

  • 임베디드할 수 있습니다. 모든 vert.x 플랫폼에서 작동하는걸 원치 않으면 라이브러리 형식으로 Java나 Groovy 어플리케이션에 집어 넣어버릴수도 있습니다.

  • 문서에서 다른 더 많은 특징들을 살펴보세요.

어디 코드를 한번 봅시다.


확장성이 뛰어난 웹 서버를 만들어 봅시다. 이 서버는 webroot 디렉토리에 있는 파일을 보여줍니다.

(아래의 탭들 중에서 각각의 언어들을 클릭해보세요.)

  • JavaScript
  • Ruby
  • Python
  • Groovy
  • Java

다음 코드를 server.js로 저장합니다.

load('vertx.js')

vertx.createHttpServer().requestHandler(function(req) {
    var file = req.path === '/' ? 'index.html' : req.path;
    req.response.sendFile('webroot/' + file);
}).listen(8080)

그리고 다음과 같이 실행합니다.

vertx run server.js

시스템의 여러 core들을 모두 사용하고 싶다구요?

그럼 이렇게 실행해 봅시다.

vertx run server.js -instances 32

Vert.x 은 다수의 인스턴스에서 동작할 것입니다.

다음 코드를 server.rb로 저장합니다.

require "vertx"

Vertx::HttpServer.new.request_handler do |req|
    file = req.uri == "/" ? "index.html" : req.uri
    req.response.send_file "webroot/#{file}"
end.listen(8080)

그리고 다음과 같이 실행합니다.

vertx run server.rb

시스템의 여러 core들을 모두 사용하고 싶다구요?

그럼 이렇게 실행해 봅시다.

vertx run server.rb -instances 32

Vert.x 는 다수의 인스턴스에서 동작할 것입니다.

다음 코드를 server.py로 저장합니다.

import vertx

server = vertx.create_http_server()

@server.request_handler
def request_handler(req):
    file = "index.html" if req.uri == "/" else req.uri
    req.response.send_file("webroot/%s"%file)
server.listen(8080)

그리고 다음과 같이 실행합니다.

vertx run server.py

시스템의 여러 core들을 모두 사용하고 싶다구요?

그럼 이렇게 실행해 봅시다.

vertx run server.py -instances 32

Vert.x 는 다수의 인스턴스에서 동작할 것입니다.

다음 코드를 Server.groovy로 저장합니다.

vertx.createHttpServer().requestHandler { req ->
    def file = req.uri == "/" ? "index.html" : req.uri
    req.response.sendFile "webroot/$file"
}.listen(8080)

그리고 다음과 같이 실행합니다.

vertx run Server.groovy

시스템의 여러 core들을 모두 사용하고 싶다구요?

Vert.x 는 다수의 인스턴스에서 동작할 것입니다.

다음 코드를 Server.java로 저장합니다.

import org.vertx.java.core.Handler;
import org.vertx.java.core.http.HttpServerRequest;
import org.vertx.java.deploy.Verticle;

public class Server extends Verticle {
    public void start() {
        vertx.createHttpServer().requestHandler(new Handler<HttpServerRequest>() {
            public void handle(HttpServerRequest req) {
                String file = req.path.equals("/") ? "index.html" : req.path;
                req.response.sendFile("webroot/" + file);
            }
        }).listen(8080);
    }
}

그리고 다음과 같이 실행합니다.

vertx run Server.java

시스템의 여러 core들을 모두 사용하고 싶다구요?

그럼 이렇게 실행해 봅시다.

vertx run Server.java -instances 32

Vert.x 는 다수의 인스턴스에서 동작할 것입니다.
